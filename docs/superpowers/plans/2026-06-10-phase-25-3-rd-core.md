# Phase 25.3 — rd_core redisplay engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The portable redisplay engine: buffers (gap buffer), a window tree, faces, glyph-matrix layout with cell-diff damage, and font painting — dual-built like `lm_core` (KTEST red-greens layout/damage in-kernel; `/bin/lisp` links it in 25.4).

**Architecture (glyph matrices, the curses/Emacs double-buffer):** redisplay = (1) LAYOUT the model into a back grid of cells `{ch, face}` (160×45 at 1280×720 with 8×16 cells — the recovered 8×8 font painted with each row doubled), (2) DIFF against the front grid → changed cells → merged damage rects, (3) PAINT changed cells into the pixel framebuffer, copy back→front. Damage minimality falls out of the diff for free. `rd_core` never calls Lisp and never calls the kernel: pixels go into a caller-supplied buffer, damage rects come back to the caller (who gfx_flushes them).

**Files:** Create `src/rd.h`, `src/rd_core.c` (font: `src/font8x8.h`, recovered from `35f4c07`). Modify `src/tests.c`.

**Model (in rd.h):**

```c
#define RD_CELL_W 8
#define RD_CELL_H 16              // 8x8 font, rows painted twice (VGA style)
#define RD_MAX_WIN 16             // window-tree node pool
#define RD_NFACES 8
struct rd_face { uint32_t fg, bg; };
struct rd_cell { unsigned char ch; unsigned char face; };
struct rd_buffer {                 // gap buffer + point
    char name[32];
    char *text; int cap, gap_start, gap_end;   // [0,gap_start)+[gap_end,cap)
    int point;                     // insertion position (in text coordinates)
    int kind;                      // RD_TEXT now; RD_SURFACE reserved (25.6)
};
struct rd_win {                    // one node: split OR leaf
    int used, leaf;
    int vertical; float would-be-ratio -> int ratio_pct;   // split
    struct rd_win *a, *b, *parent;
    struct rd_buffer *buf; int top_line;                   // leaf
};
struct rd_frame {
    int px_w, px_h, cols, rows;
    struct rd_win wins[RD_MAX_WIN];
    struct rd_win *root, *selected;
    struct rd_face faces[RD_NFACES];      // 0=default 1=modeline 2=region...
    char echo[256];
    struct rd_cell *front, *back;          // cols*rows each (caller-allocated)
    int cursor_col, cursor_row;            // computed by layout
};
```

**API:** `rd_buf_init/insert/delete/point_set/textlen/char_at`,
`rd_frame_init(frame, w, h, front, back, buf0)`, `rd_split(frame, vertical)`,
`rd_win_delete(frame)`, `rd_other_window(frame)`, `rd_set_buffer(frame, buf)`,
`rd_layout(frame)`, `rd_cell_at(frame, col, row)`,
`rd_redisplay(frame, fb, stride_px, rects[], max) -> n_damage`,
`rd_echo(frame, str)`.

**Layout rules:** window rect in cells; text lines truncate (no wrap in v1);
last cell row of every window = modeline (`-- <bufname> --` padded, face 1);
frame's last row = echo area; vertical-split border = the split's first column
of window b... no border column in v1: splits land on cell boundaries, the
modelines delineate. Cursor = cell of selected window's point.

## Tasks (each: red KTEST → code → green → commit)

1. **Gap buffer** — `rd: gap buffer insert/delete/read` (insert "hello",
   delete, char_at across the gap, point moves).
2. **Single-window layout** — `rd: single window layout` (frame 640x320 → 80x20
   cells; buffer "hi" → cells (0,0)='h' (1,0)='i'; modeline row 18 face 1 with
   buffer name; echo row 19; long line truncates at col 79).
3. **Splits** — `rd: split below + other window` (split → window a rows 0..9,
   b rows 10..18, both with modelines; selected toggles; set-buffer changes the
   selected leaf only). `rd: split right` (columns halved).
4. **Damage diff** — `rd: damage confined to edited window` (redisplay all,
   then insert into one buffer → all damage rects within that window's pixel
   rect; second redisplay with no change → 0 rects; cursor move alone → ≤2 rects).
5. **Paint** — `rd: glyphs hit the framebuffer` (redisplay into a small pixel
   buffer; 'A' cell has fg pixels set per font8x8['A'], bg elsewhere; modeline
   cells use face-1 colors; cursor cell inverted).
6. Commit per task; suite green throughout.

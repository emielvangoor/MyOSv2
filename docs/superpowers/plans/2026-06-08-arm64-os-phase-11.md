# MyOSv2 Phase 11 — ASIDs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Tag TLB entries per address space so context switches no longer flush the whole TLB.

**Architecture:** Each `struct addrspace` gets a 16-bit ASID from a monotonic allocator (ID 0 reserved). User page descriptors become non-global (nG=1) so they're ASID-tagged; kernel mappings stay global. `as_switch` loads the ASID into `TTBR0_EL1[63:48]` and skips the flush. Targeted `tlbi aside1` replaces `tlbi vmalle1` when a single live address space's mappings change (COW); a full flush remains only for ASID rollover. `TCR_EL1.AS=1` enables 16-bit ASIDs.

**Tech Stack:** ARM64 AArch64, C + inline asm, QEMU virt, in-kernel KASSERT tests via `make test`.

Reference: `docs/superpowers/specs/2026-06-08-arm64-os-phase-11-design.md`.

```c
static void test_asid_clone_distinct(void)
{
    pmm_init(); kheap_init(); vm_init();
    struct addrspace *p = as_create();
    struct addrspace *c = as_clone(p);
    KASSERT(c->asid != p->asid);       // a clone is its own address space

static void test_asid_user_page_nonglobal(void)
{
    pmm_init(); kheap_init(); vm_init();
    struct addrspace *a = as_create();
    uint64_t *pte = as_pte(a, USER_DATA_VA);
    KASSERT(pte != 0);
    KASSERT((*pte & (1UL << 11)) != 0);   // nG set -> ASID-tagged
}

static void test_asid_rollover_recycles(void)
{
    vm_init();                            // next_asid = 1
    uint16_t first = asid_alloc();        // 1
    for (uint32_t i = 1; i < 0xFFFF; i++) { asid_alloc(); }   // consume up to ASID_MAX
    uint16_t wrapped = asid_alloc();      // ASID_MAX+1 -> wrap
    KASSERT(first == 1);
    KASSERT(wrapped == 1);                // recycled from the bottom
}
```

- [ ] **Step 4: Implement in `src/vm.c`**

(a) Add the nG bit constant near the other descriptor bits (after `ATTR_PXN`):

```c
#define ATTR_NG         (1UL << 11)   // non-global: entry is tagged by the current ASID
```

(b) Add the ASID allocator and full-flush helper. Place the counter near
`page_ref` and define `flush_all_tlb` before `asid_alloc` uses it:

```c
// ---- ASIDs: tag TLB entries per address space (Phase 11) ----
#define ASID_MAX 0xFFFF
static uint32_t next_asid = 1;        // 0 is reserved (boot/unused TTBR0 value)

static void flush_all_tlb(void)       // drop EVERY EL1 TLB entry (rollover only)
{
    __asm__ volatile("dsb ish");
    __asm__ volatile("tlbi vmalle1");
    __asm__ volatile("dsb ish");
    __asm__ volatile("isb");
}

uint16_t asid_alloc(void)
{
    if (next_asid > ASID_MAX) {       // handed out every ID -> recycle from 1
        next_asid = 1;
        flush_all_tlb();              // clear any dead space's entries under reused IDs
    }
    return (uint16_t)(next_asid++);
}
```

(c) In `vm_init()`, reset the counter alongside the refcounts:

```c
void vm_init(void)
{
    for (uint64_t i = 0; i < NPAGES; i++) { page_ref[i] = 0; }   // reset refcounts
    next_asid = 1;                                               // reset ASID allocator
}
```

(d) Add `ATTR_NG` to every user-page attribute set so user mappings are tagged.
In `as_create_image`:

```c
    uint64_t code_attr = ATTR_AF | ATTR_SH_INNER | ATTR_IDX_NORMAL | AP_RO_ALL | ATTR_NG;  // RO, EL0-exec
    uint64_t rw_attr   = ATTR_AF | ATTR_SH_INNER | ATTR_IDX_NORMAL | AP_RW_ALL |
                         ATTR_UXN | ATTR_PXN | ATTR_NG;                                     // RW, no-exec
```

Set its ASID right after building `as->l0`:

```c
    struct addrspace *as = (struct addrspace *)pmm_alloc();
    as->l0 = alloc_table();
    as->asid = asid_alloc();
```

(e) `as_switch` carries the ASID and drops the flush:

```c
void as_switch(struct addrspace *as)
{
    uint64_t ttbr = ((uint64_t)as->asid << 48) | (uint64_t)(uintptr_t)as->l0;
    __asm__ volatile("msr ttbr0_el1, %0" :: "r"(ttbr));
    __asm__ volatile("isb");
    // No TLB flush: the ASID keeps the previous space's entries from matching.
}
```

(f) Replace `flush_tlb()` with a per-ASID version. Change its definition (in the
COW section) to:

```c
static void flush_tlb_asid(uint16_t asid)   // invalidate only this ASID's entries
{
    uint64_t arg = (uint64_t)asid << 48;
    __asm__ volatile("dsb ish");
    __asm__ volatile("tlbi aside1, %0" :: "r"(arg));
    __asm__ volatile("dsb ish");
    __asm__ volatile("isb");
}
```

(g) In `as_clone`: set the child's ASID, add `ATTR_NG` to its attr sets, and flush
the parent's ASID. Change the top:

```c
    struct addrspace *child = (struct addrspace *)pmm_alloc();
```

Change its attr constants:

- [ ] **Step 5: Enable 16-bit ASIDs in `src/mmu.c`**

In `enable_mmu()`, add the `AS` bit to the `tcr` value (bit 36 = 16-bit ASID):

```c
    uint64_t tcr =
        (16UL << 0)  |   // T0SZ=16  -> TTBR0 covers a 48-bit VA range (2^48)
        (1UL  << 8)  |   // IRGN0=WB -> page-table walks use write-back cache
        (1UL  << 10) |   // ORGN0=WB
        (3UL  << 12) |   // SH0=inner shareable
        (0UL  << 14) |   // TG0=0    -> 4 KiB granule
        (1UL  << 23) |   // EPD1=1   -> disable TTBR1 (we only use the low/TTBR0 half)
        (2UL  << 32) |   // IPS=2    -> 40-bit physical address size
        (1UL  << 36);    // AS=1     -> 16-bit ASIDs (TTBR0[63:48])
```
